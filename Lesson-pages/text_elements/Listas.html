In the above list section we used the <style> ... </style> element to open a (probably) CSS field in the code and define styles using "#string-of-the-id" and ".string-of-the-class" which were attached int the "list" and "item" elements. Every element tagged with id="string" or class="string" will follow the determined style to each class and id.
Element ids are supposed to be unique in the HTML document, then using element classes are rather useful when configuring multi-elements styles. Moreover, beside CSS, JavaScript may also refer to specific HTML elements through their classes and ids. That's also why we should keep our ids and classes coherently organized.

The "li" value attribute, <li value="ranking">

Sometimes we may want to define our list elements ranks precisely or arbitrarily in our ordered lists. In those cases we add the "value" tag to the "li" element. Just inside the <li> opening tag element. We may also use the "start" definition to make our list start at the 60th rank.

  • <ol start="60">
    1. 60th rank
    2. 61st rank
    3. <li value="69">
    4. 70th rank
    5. nth rank
  • </ol>
