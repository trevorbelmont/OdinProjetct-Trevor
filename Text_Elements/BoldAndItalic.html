"Strong" Element (bold imporant texts)

The strong element, <strong> ... </strong>, makes the text in between the strong tags bold. This also confers importance to the text, which maybe used to modolute the tone of voice by screen readers which aid people with visual impairments or disabilities. It's also possible to make some text bold without making them more imporant through CSS - but I just didn't started CSS in Odin Project curriculum yet.
Strong element may even be used on its own (without a text container as paragraph). But it's a practice to be avoid and also requires unnecessary efforts to break lines, format and adapt to screens.
The line below is an example of use of <strong> tags without a text container.

  "Este é um texto avulso e usa negreito (strong tag)."
As you can check in this page's source code, an extra break line ( <br> ) and a tabulation space ( &emsp; ) was needed in order to keep the line above separate from the previous and current paragraph and format it.

"Em" Element ( italic emphasized texts)

Similarly we can use <em> ... </em>, to emphasize some parts of our texts by making it "italic". Just like the <strong> tag, an </m> tag may impact users who are using any type of screen-readers or reading aid. That's because, since italic texts are used to emphasize some information or text fragments in printed texts, these tags may be used to modulate voice or something in order to sonically emphasize the text in between </em> tags.
</em> tags may also be used alienated from the classics containers - just like <strong> tags:
