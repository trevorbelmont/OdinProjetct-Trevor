O elemento de "header 1", <h1>, acima foi especificado com " lang = "en-GB" " (Great Britain).

 Este é um elemento "text". Text elements exibem elementos textuais em seguida e na mesma linha. Elementos "text" também não possuem quebras de linhas. Portanto todos são sequencionados na msm linha.

A não ser que haja abertura e fechamento de um elemento parágrafo dentro deles: <p> ... </p > ...

Este text element foi especificado com " lang = "pt-BR" "

Escape characters

Escape characters são feitos entre '&' e ';'

Exemplo: para o sinal de "menor" e maior usamos < > (less than & greater than)

A "e comercial", &, é feita com "& + amp + ;" (tudo junto). "Amp" comes from "ampersand" (&)

Elementos "paragraph"

Usar "<p>" (elemento "paragraph") é mais simples e útil que usar o elemento "text".

Apesar de que o elemento <p> pode ser usado sem precisar fechar dentro de outros elementos de texto, é mais útil usá-lo para escrever. Ele é mais prático que o elemento "text". Além disso os VSCode reconhece o uso de um elemento <p> sem fechar como erro.

Outra vantagem usar <p> para elementos escritos é que o elemento "paragraph" ajusta o texto automaticamente às telas e ao redimensionamento das telas.

Este parágrafo está defindido em português brasileiro. Igualmente podemos definir os elementos "paragraphs" em qualquer língua. E se quisermos podemos definir também regiões como "-BR", "-en-GB" e etc., porém regiões não tem impacto no HTML. Especificar regiões PODE ser útil pois estas podem ser passadas para plugins habilitados nos navegadores.
