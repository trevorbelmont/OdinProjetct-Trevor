Handling HyperLinks and Images in HTML

This lesson-page exercise handles how to use <a> to turn any element or fragment of element (e.g. parts of a texts) into an anchor to another content (page, site, video, img or whatsoever). We are also going to learn how to use the "href" attribute to point and direct these anchors to specific locations (url).
We are also going to learn how to display images in our sites, be it from our own domain or from another's. Simply combining those techniques we must be able to turn an image an hyperlink too.


Anchors and Hyper-References : <a href="address">

The <a> ... </a>, "anchor" elements tags, turns almost every content element between these tags into an anchor to another page or content. However, just delimiting an anchor with some anchor tags are not enough to point that anchor to any other page or content. Thus, creating only an empty anchor will lead us to nowhere else through a pointless hyperlink.
The target="_blank" attribute setting in the "anchor" tag defines whether the link shall be open in a new blank tab.

The "href" attribute

In order to point an anchor content, <a>, to its actual destination we add the "href" attribute to the anchor element opening tag. This way - be it an image, a text, a button or anything - the anchored element will become a clickable visual shortcut to the referenced content address.

Absolute and Relative Links in "href"

Depending on whether we're pointing to an address inside or outside our page domain, we will have to use two different types of hyper-referencing - absolute and relative.

  • Absolute Links
     Those are the ones we use to link our content element to a page or address from another domain other than the linking page current domain. Absolute links are supposed to always follow the "protocol://domain/path" pattern. Preferably we must chose "https" protocol over "http" when both are available. "Domain" is literally self-explanatory and "path" is the rest of the address which leads us to the exactly page or content in the given domain that our anchor element is anchored to. E.g.:   "https:// + techtarget.com/ + ridiculous-long-path"
  • Relative Links
     On the other hand, whenever we want to link our elements to another page within our own domain we simply use relative links. Relative links are relative to our domain index and are pointed quite similarly to how we point folders on terminal and powershell: mostly using directories names, bars and typing file extensions. "protocol://domain/path" pattern. Preferably we must chose "https" protocol over "http" when both are available. "Domain" is literally self-explanatory and "path" is the rest of the address which leads us to the exactly page or content in the given domain that our anchor element is anchored to. E.g.:   "https:// + tutorialspoint.com/ + awful-path"

  4 Ways of Indicating Relative Paths

  • "file.extension" : Used when linking to content in the same folder.
    • "folder/anotherFile.extension" : Links to the given file content in the given folder path from the current directory forward.
    • " ./folder/anotherFile.extension" : Does exactly the same as above but with less risks of crashing.
  • (root)/folder/someFile.extension" : Points content relative to the index location. The "/" at the beginning signifies the domain's root folder.
  • (parent)../otherFolder/anyFile.extension : Use "../" to ascend to the parent folder and navigate the folder from there on.

Images, Source and Alt. Descriptions: <img src="adress" alt="description">

In order to make our site more attractive or achieve specific goals we may probably want to add pictures to our pages.Thus we shall use the self-closing <img> element. While HTML has a great variety of elements to handle different types of media contents, the "img" is surely one of the most used. A "img" element must perform 3 types of actions: be the container to the picture media, point the source of the image and provide an alternative description of its image to the ones who can't see or interpret the image visual information. Hence we are going to use at least two attributes in the "img" tag element:

< img src = "address" alt = "description" >

  • <img>, the image container itself. A self-closing tag.
  • src="address", attribute which points the relative or absolute path address of the media.
  • alt="description", attribute which specify the image description in natural language.
  • width = "number", "percentage %", "pixels px", "auto"
  • height = "number", "percentage%", "pixelspx", "auto"
  • max-width: 120%

In addition to the more common "png" and "jpg" web image file extensions. "svg" (Scalable Vector Graphics) and "gif" (Graphic Interchange Format) are also quite recurrent and main formats when building or surfing pages on internet.
